INSTRUÇÃO OFICIAL PARA O FRONTEND
Projeto: Fluxo de Caixa RA
Direção visual: NASA Mission Control / ERP retrô técnico / terminal financeiro 80s-90s com execução moderna
Objetivo

Redesenhar completamente o visual do sistema para transmitir a sensação de:

painel de controle técnico
centro de comando financeiro
software corporativo antigo refinado
interface inspirada em NASA / sistemas industriais / monitores de operação
nostalgia 80s/90s sem comprometer usabilidade moderna

O sistema não deve parecer “velho e mal feito”.
Ele deve parecer retro-técnico, robusto, confiável, funcional e memorável.

A sensação desejada é:

“um sistema financeiro que parece operar uma missão crítica”

1. PRINCÍPIO GERAL DE DESIGN
O que queremos

Criar uma interface com identidade forte, misturando:

estética de sistemas antigos
dashboards de missão
terminais técnicos
linguagem visual de monitoramento
UX moderna e limpa
O que não queremos

Não fazer:

visual caricatural
pixel art exagerado
efeito CRT pesado em tudo
interface desconfortável
fonte difícil de ler
excesso de glow
excesso de nostalgia sem função
UI “trash” ou “velha de propósito”
Regra principal

A estética retrô entra em:

paleta
tipografia secundária
grids
bordas
cabeçalhos
nomenclatura
estados
microdetalhes

A usabilidade moderna entra em:

hierarquia visual
spacing
responsividade
navegação
consistência
acessibilidade
contraste
clareza de ações
2. CONCEITO VISUAL
Nome interno do tema

Mission Finance / Control Console

Referências conceituais

A equipe deve se inspirar em:

NASA Mission Control
terminais corporativos dos anos 80 e 90
monitores de operação industrial
software de contabilidade legado
consoles de comando
telas de radar, telemetria e monitoramento
painéis de controle técnico
Sensações obrigatórias

A interface final deve transmitir:

precisão
controle
operação crítica
confiabilidade
densidade informacional
sofisticação técnica
caráter industrial / operacional
3. PALETA DE CORES
Base principal

Usar fundo escuro técnico. Nada de preto puro absoluto.
Queremos um escuro levemente esverdeado/azulado.

Cores base
#0A0F0D → fundo principal
#101917 → superfícies secundárias
#15211E → cards / painéis
#1C2A27 → divisórias / blocos
Texto
#E6F1EC → texto principal
#B7C8C0 → texto secundário
#7E948A → texto discreto / metadados
Cores de destaque
#1AFF66 → sucesso / ativo / destaque terminal
#7FDBFF → informativo / dados / links / gráficos secundários
#FFB347 → alerta / atenção / status de revisão
#FF6B6B → erro / crítico / negativo
Cores auxiliares
#22342F → bordas internas
#2C4740 → hover discreto
#314C72 → áreas informativas frias
#3D3324 → fundo leve para alertas âmbar
Regras de uso
verde terminal deve ser usado com moderação, em pontos de alto valor
não usar glow forte constante
não usar saturação alta em massa
o sistema deve parecer técnico, não gamer
4. TIPOGRAFIA
Fonte principal de interface

Usar uma sans moderna, limpa e legível.

Preferência:

Inter
ou IBM Plex Sans
Fonte secundária técnica

Usar monoespaçada em pontos estratégicos.

Preferência:

IBM Plex Mono
ou JetBrains Mono
Regras
Sans:

usar para

menus
labels principais
formulários
navegação
títulos de seção
Mono:

usar para

valores monetários
saldos
datas técnicas
timestamps
códigos
status
cabeçalhos de widgets técnicos
referências de importação e logs
Escala tipográfica
Título principal: 24–32px
Título de bloco: 16–20px
Texto padrão: 14–16px
Texto técnico / label / tabela: 12–14px
Números de destaque: 24–40px
Regras adicionais
evitar peso excessivamente fino
privilegiar 500 e 600 para UI
usar caixa alta apenas em labels e headers técnicos
5. LINGUAGEM DOS COMPONENTES
Bordas

Todos os componentes devem ter bordas bem definidas.

radius baixo: 4px ou 6px
nada muito arredondado
aparência mais industrial/técnica
evitar estilo “app mobile fofinho”
Sombras
sombras discretas
profundidade baixa
nada flutuante demais
foco em separação estrutural, não efeito decorativo
Cards

Cards devem parecer painéis de controle.

Cada card precisa ter:

cabeçalho técnico
título objetivo
possível subtítulo/status
corpo limpo
divisão clara entre header e conteúdo
Tabelas

As tabelas são elemento central do sistema.

Devem ter:

cabeçalho forte
grid visível porém discreto
boa densidade
linhas com hover claro
valores alinhados
tipografia mono para números
colunas financeiras bem delimitadas
Inputs

Os campos devem passar sensação de sistema operacional técnico.

Regras:

fundo escuro secundário
borda visível
foco com cor ciano ou verde
placeholder discreto
nada transparente demais
estados bem claros: normal, foco, erro, desabilitado
Botões

Os botões devem parecer ações operacionais.

Estilo
formato reto
borda aparente
peso visual firme
sem degradês exagerados
Hierarquia
Primário: verde técnico
Secundário: cinza técnico
Atenção: âmbar
Perigo: vermelho técnico

Topbar

A topbar deve funcionar como barra de controle e contexto.

Deve conter
nome da tela atual

perfil do usuário
ações rápidas
possivelmente relógio técnico
Visual
baixa altura
informativa
elegante
com cara de console

9. TELA DE CONCILIAÇÃO
Direção

Essa deve ser uma das telas mais fortes visualmente.

Ela deve parecer uma tela de validação crítica.

Componentes sugeridos

duas colunas principais:
lançamentos internos
extrato bancário
área central ou lateral de correspondência
filtros técnicos
status de divergência
Estados

pendente
divergente
ignorado
duplicado
Visual

cores de estado muito claras
destaque moderado para diferenças
tipografia mono para datas e valores
sensação de revisão operacional
10. FORMULÁRIOS E CADASTROS
Regra

Nessas telas, reduzir o “efeito temático” e priorizar clareza.

Estrutura

botões bem evidentes
Não fazer
poluir formulário com efeitos
usar muito brilho
inventar microinterações desnecessárias

Colunas financeiras

vencimento
Cores

saídas: vermelho técnico ou âmbar quente

Proibido em excesso

desfoques desnecessários

A equipe deve validar cada tela com estas perguntas:
