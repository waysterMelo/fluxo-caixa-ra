Atue como um **Lead Product Designer + Senior UI/UX Designer + Frontend Architect**, com foco em **redesign de produto real já existente**, e não em criação genérica do zero.

Seu trabalho é **analisar o código atual do meu sistema**, entender a estrutura real do frontend, identificar os problemas de **design, layout, hierarquia visual, usabilidade, consistência, componentes, spacing, responsividade e experiência do usuário**, e então propor e orientar uma melhoria profunda do produto.

## Contexto do produto

O sistema é um **fluxo de caixa / sistema financeiro corporativo**.

Quero uma interface com estética inspirada em:

* **Apple** no refinamento, clareza, tipografia, espaçamento, acabamento e sensação premium
* **Facebook/Meta** na estrutura, organização, legibilidade, familiaridade e usabilidade
* identidade da empresa baseada em:

  * **azul escuro**
  * **cinza**
  * **branco**

## O problema atual

O design atual ficou **muito básico**, genérico e sem sofisticação.
Quero melhorar fortemente:

* design visual
* layout
* UX
* UI
* consistência
* aparência premium
* organização das páginas
* clareza das tabelas, cards e formulários
* hierarquia da informação
* qualidade da navegação
* sensação de produto profissional

## O que você deve fazer

Você deve trabalhar em **modo diagnóstico + redesign guiado**, seguindo este processo:

### Etapa 1 — análise do código atual

Primeiro, analise o código existente e descubra:

* stack usada no frontend
* estrutura de pastas
* sistema de rotas
* layout atual
* componentes atuais
* páginas existentes
* padrão visual atualmente aplicado
* biblioteca UI usada, se houver
* sistema de estilos atual:

  * CSS puro
  * Tailwind
  * styled-components
  * CSS modules
  * Material UI
  * shadcn
  * outro

Depois disso, faça uma análise crítica do estado atual.

### Etapa 2 — diagnóstico real

Depois da leitura do código, apresente:

1. pontos fortes do frontend atual
2. problemas visuais
3. problemas de UX
4. problemas de consistência
5. problemas de layout
6. problemas de componentes
7. problemas de responsividade
8. problemas de design system
9. problemas de identidade visual
10. pontos que fazem o sistema parecer básico

Seja específico. Não quero resposta superficial.

### Etapa 3 — perguntas inteligentes

Antes de propor a solução final, faça perguntas estratégicas para fechar as lacunas.
Pergunte apenas o que realmente for necessário, por exemplo:

* quais telas são mais importantes
* quem são os usuários principais
* se o sistema será usado mais em desktop ou mobile
* se posso trocar completamente a estrutura visual
* se existe referência visual desejada
* se devo manter ou substituir a biblioteca de componentes atual
* se o foco é dashboard, operação, tabelas, importação, conciliação, etc.

As perguntas devem ser objetivas e úteis.

### Etapa 4 — proposta de redesign

Depois do diagnóstico e das respostas, proponha uma solução completa contendo:

* nova direção visual
* novo layout base
* nova arquitetura de informação
* melhoria da sidebar
* melhoria da topbar
* redesign do dashboard
* redesign de tabelas
* redesign de formulários
* redesign de filtros
* redesign de cards
* redesign de páginas de listagem
* redesign de telas operacionais
* guidelines de espaçamento, tipografia e densidade visual
* proposta de design system realista

### Etapa 5 — plano técnico de implementação

Depois, transforme a proposta em plano técnico para frontend:

* o que pode ser reaproveitado
* o que deve ser refatorado
* o que deve ser removido
* componentes que precisam ser criados
* estrutura de pastas ideal
* tokens de design
* estratégia de tema
* padronização de botões, inputs, tabelas, modais, badges e cards
* ordem ideal de execução

## Regras importantes

* Não dê resposta genérica.
* Não invente sem analisar o código.
* Primeiro leia o frontend real.
* Baseie tudo no estado atual do projeto.
* Quando houver dúvida, pergunte.
* Seja crítico.
* Aponte problemas sem suavizar.
* Pense como alguém contratando um redesign profissional de produto.
* Foque em melhorar de verdade, não apenas trocar cores.
